# One-API 分组与限流关系说明

## 核心结论

**分组（Group）类型不影响 API 并发速度限制**。所有分组共享相同的全局限流配置。

## 详细说明

### 1. 分组（Group）的作用

分组主要用于以下两个方面：

#### ① 渠道选择
- 不同分组的用户可以使用不同的渠道池
- 渠道可以配置支持哪些分组（在渠道设置中的"分组"字段）
- 例如：`svip` 分组的用户只能使用标记为 `svip` 分组的渠道

**代码位置**：`middleware/distributor.go:47`
```go
channel, err = model.CacheGetRandomSatisfiedChannel(userGroup, requestModel, false)
```

#### ② 计费倍率（GroupRatio）
- 不同分组可以设置不同的计费倍率
- 默认所有分组的倍率都是 1.0
- 可以在系统设置中配置分组倍率（JSON 格式）

**代码位置**：`relay/billing/ratio/group.go`
```go
var GroupRatio = map[string]float64{
    "default": 1,
    "vip":     1,
    "svip":    1,
}
```

**计费公式**：
```
最终消耗额度 = 分组倍率 × 模型倍率 × (提示 token 数 + 补全 token 数 × 补全倍率)
```

### 2. 限流（Rate Limiting）机制

#### 限流是基于 IP 地址的全局限制

**限流配置**（`common/config/config.go`）：
- `GLOBAL_API_RATE_LIMIT`: 全局 API 限流，默认 **480 次/3分钟**（单 IP）
- `GLOBAL_WEB_RATE_LIMIT`: 全局 Web 限流，默认 **240 次/3分钟**（单 IP）

**限流实现**（`middleware/rate-limit.go`）：
- 限流 key 基于 IP 地址：`rateLimit:GA{IP地址}`
- **所有用户共享相同的限流配置**，不管分组是什么
- 限流是全局的，不是按分组分别限流

#### 限流速度对比

| 限流类型 | 默认值 | 说明 |
|---------|--------|------|
| **GLOBAL_API_RATE_LIMIT** | **480 次/3分钟** | **最大速度**，用于 API 请求 |
| GLOBAL_WEB_RATE_LIMIT | 240 次/3分钟 | Web 界面请求 |
| CriticalRateLimit | 20 次/20分钟 | 关键操作（如登录） |
| UploadRateLimit | 10 次/分钟 | 文件上传 |
| DownloadRateLimit | 10 次/分钟 | 文件下载 |

**结论**：`GLOBAL_API_RATE_LIMIT` 是最大的限流速度，默认 480 次/3分钟 = **160 次/分钟**。

### 3. 分组与限流的关系

#### ❌ 分组不影响限流速度

- 所有分组（`default`、`vip`、`svip` 等）共享相同的限流配置
- 限流是基于 IP 地址的，不是基于用户分组的
- 即使设置了不同的分组，限流速度都是一样的

#### ✅ 分组只影响

1. **渠道选择**：不同分组可以使用不同的渠道
2. **计费倍率**：不同分组可以有不同的计费倍率（但不影响请求速度）

### 4. 如何提高 API 请求速度

如果想提高 API 请求速度，可以：

#### 方法 1：调整全局限流配置

通过环境变量调整：
```bash
# 提高 API 限流（例如设置为 1000 次/3分钟）
export GLOBAL_API_RATE_LIMIT=1000
```

#### 方法 2：使用多个 IP 地址

由于限流是基于 IP 的，使用不同的 IP 地址可以绕过单 IP 限流。

#### 方法 3：禁用限流（不推荐）

在开发环境中可以禁用限流：
```bash
export DEBUG=true  # 调试模式下会禁用限流
```

### 5. 实际应用场景

#### 场景 1：不同分组使用不同渠道
- `default` 分组 → 使用普通渠道
- `svip` 分组 → 使用高质量渠道（如 Gemini）

#### 场景 2：不同分组不同计费
- `default` 分组 → 倍率 1.0（正常计费）
- `vip` 分组 → 倍率 0.8（8折）
- `svip` 分组 → 倍率 0.5（5折）

#### 场景 3：限流控制
- 所有分组共享相同的限流：480 次/3分钟
- 如果需要不同的限流速度，需要修改全局配置或使用不同 IP

## 总结

1. **分组不影响限流速度**：所有分组共享相同的限流配置
2. **限流速度最大**：`GLOBAL_API_RATE_LIMIT`，默认 480 次/3分钟（160 次/分钟）
3. **分组的作用**：
   - 渠道选择（不同分组可用不同渠道）
   - 计费倍率（不同分组可设置不同倍率）
4. **提高速度的方法**：
   - 调整 `GLOBAL_API_RATE_LIMIT` 环境变量
   - 使用多个 IP 地址
   - 在开发环境启用 DEBUG 模式

## 相关配置

### 环境变量
```bash
# API 限流（最大速度）
GLOBAL_API_RATE_LIMIT=480  # 默认值，可调整

# Web 限流
GLOBAL_WEB_RATE_LIMIT=240  # 默认值
```

### 系统设置（后台）
- **分组倍率**：在系统设置中可以配置不同分组的计费倍率
- **渠道分组**：在渠道设置中可以配置渠道支持的分组
